### 当前代码现状（必须诚实计入排期）

截至 2026-07-13，仓库已经有 OpenAI-compatible API 调用、因子提议/筛选 prompt 和 `llm-interpret`。但是 `llm-interpret` 的用途是让模型解释或改写公式，**不是可解释性评测器**。

以下论文关键设施目前均未实现：

- 盲评公式包导出；
- 带评分 rubric 和变量元数据的 LLM judge prompt；
- 1–5 分结构化响应 schema 与严格校验；
- 多模型批量调用、缓存、重试和失败日志；
- 公式顺序/左右位置扰动；
- 人类评分录入模板；
- 人类与 LLM 评分聚合、一致性和置信区间统计。

因此，“实现并校准可解释性评测代码”是 **P0 前置开发**，不能等到 08-06 开始盲评时才做。

## 0. 先给结论

这项工作适合按一篇**方法 + 实证评测型的 CCF C 论文**来做，主线不需要改变：

> 真实科学数据通常有噪声、样本有限且关系复杂，因此唯一真实公式往往不可辨识。此时符号回归的现实价值，不应只被定义为复原真公式，而应是在预测性能可接受的前提下，产生人类能够理解、审查并用于提出假设的表达式。现有符号回归多以预测误差和语法复杂度为主要目标，但低节点数并不等于领域可解释。本文提出一个可解释性优先的符号回归流程，在接近最优预测性能的候选中，用领域因子、语义约束和展开后的真实复杂度选择表达式。

论文不写“现有方法完全没有可解释性约束”，而写：

> 现有方法已有复杂度、单位、稳定性及 LLM 辅助等尝试，但主流方法采用的语法复杂度仍不足以表示真实科学任务中的领域语义可解释性。

快速发表的关键不是再造一个大型 SR 搜索引擎，而是把以下闭环做扎实：

1. 定义一个可复现的 interpretability-first 流程；
2. 在无真式的真实黑盒任务上证明：R² 基本不降，人和独立 LLM 更偏好我们的公式；
3. 在有复杂真式的任务上证明：除上述结果外，表达式与真式更接近；
4. 所有比较采用无泄漏 split、相同计算预算和盲评。

## 1. 工作的价值与论文贡献

### 1.1 工作价值

- **重新定义真实数据上的成功标准**：真实数据上无法验证“发现了自然定律”，但可以验证公式是否预测有效、是否容易理解、是否更接近已知结构。
- **补上预测与解释之间的评测缺口**：只报 R² 会奖励复杂但不可读的公式，只报节点数又无法表示变量组合是否有领域意义。
- **给出可落地的选式原则**：不是把准确率和解释性粗暴加权，而是先规定准确率容忍带，再在带内优先解释性。
- **建立双轨证据**：真实黑盒数据验证实际用途；已知真式数据验证结构恢复能力。

### 1.2 最小可发表贡献

论文只承诺三个贡献：

1. **流程贡献**：一个 accuracy-constrained、interpretability-first 的候选生成和选式流程；
2. **评价贡献**：R² + 人类专家盲评 + 独立 LLM 盲评的真实黑盒评测协议；
3. **验证贡献**：在已知复杂表达式任务上加入统一的表达式相似度，验证流程不仅“看起来简单”，而且更接近生成结构。

这组贡献若实验完整、无泄漏且优势清楚，足够支撑 CCF C Full Paper。若只得到少数案例或只有主观展示，则不够。CCF B 可以作为结果显著且方法创新超出 reranking 时的后续目标，不建议为冲 B 错过当前窗口。

## 2. 方法最小版本

暂定名称：**IF-SR（Interpretability-First Symbolic Regression）**。

### 2.1 流程

```text
训练数据 + 变量元信息
        ↓
原始变量 + 数据挖掘因子 + 有记录的领域因子
        ↓
符号回归搜索，保留完整候选 Pareto archive
        ↓
在 validation 上建立准确率容忍集合
R²_val ≥ best R²_val - δ，暂定 δ=0.02
        ↓
剔除硬语义违规
        ↓
按展开后复杂度、领域因子可解释性和稳定性选式
        ↓
锁定公式后，仅在 test 上评价一次
```

领域因子不是隐藏复杂度的别名。每个因子必须保存名称、原始变量表达式、含义、单位状态和来源；最终公式必须完全展开到原始变量后再计算复杂度。

### 2.2 选择规则

P0 版本使用简单、可复现的词典序规则：

1. validation R² 进入最优值的 `δ=0.02` 容忍带；
2. 剔除非法变量、量纲加减冲突、无定义或有效域覆盖不足的表达式；
3. 最小化完全展开后的节点数；
4. 同复杂度时，优先选择包含经确认领域因子、跨 seed 更稳定的公式。

不把论文评价所用的 LLM judge 直接放进选择目标，避免“用同一个裁判选答案再证明答案得分高”的循环论证。

## 3. 两个核心实验

### 3.1 实验 1：无明确真式的真实黑盒数据

研究问题：

> 在真实噪声数据上，IF-SR 能否在基本保持 test R² 的同时，得到人类专家和独立 LLM 更容易理解的表达式？

快速版数据范围：

- 主领域保持催化/材料，不扩到多个领域；
- 先从现有 targets 中冻结 **6 个代表任务**作为主表；
- 选择理由必须在看最终方法结果前写定，覆盖不同样本量、可预测性和 target 类型；
- 其余现有 targets 只在计算自动化完成后放入补充材料；
- 同一源表的多个 target 不能声称为多个独立数据集。

比较方法固定为：

1. `PySR`：标准复杂度—误差选择；
2. `SISSO++`：外部符号回归 baseline；若在 pilot 中确实无法统一运行，换 Operon；
3. `Mine-PySR`：数据挖掘因子 + PySR，不用领域语义选择；
4. `IF-SR`：完整流程；
5. `IF-SR w/o domain factors`：只做可解释性优先选式，用于证明领域因子的作用。

必须保证相同数据 split、近似相同 wall-clock 或 evaluation budget。每个任务每种方法 3 seeds；若资源允许，6 个主任务提升到 5 seeds。

主评价标准严格沿用论文主线：

- **预测性**：独立 test R²；NRMSE 作为补充；
- **人类可解释性**：领域专家盲评得分；
- **LLM 可解释性**：两个不同模型家族的独立盲评得分；
- **辅助诊断**：展开节点数、变量数、失败率和运行时间。

### 3.2 实验 2：有明确复杂真式的数据

研究问题：

> 当生成真式已知时，IF-SR 是否在保持 R² 和可解释性优势的同时，更接近真实表达式？

快速版范围：

- 从一个公开 benchmark 冻结 **20 个中高复杂度公式**；
- 避免只挑本方法擅长的公式，按变量数、节点数和算子分层抽取；
- 主实验使用 `5% target noise + 5 个无关变量`；
- clean 结果作为补充；
- 每题每种方法 3 seeds；
- 隐藏公式名称和领域描述，随机重命名变量，避免 LLM 直接记忆答案。

评价标准：

- test R²；
- 与实验 1 相同的人类/LLM 可解释性分；
- 精确或数值等价恢复率；
- 一个预先固定的表达式相似度总分。

快速版表达式相似度定义为：

```text
ExprSim = 0.50 × 独立采样点上的数值相似度
        + 0.20 × 变量集合 F1
        + 0.20 × 运算符集合 F1
        + 0.10 × 规范化树结构相似度
```

所有子分数归一化到 `[0,1]`。代数等价成功率单独报告，不能只靠综合分掩盖未恢复真式。权重在确认性实验前冻结，不根据结果修改。

## 4. 可解释性评分协议

为了赶 8 月 21 日，主观评价采用“小而严格”，不做大问卷。

### 4.1 评分维度

每个公式按 1–5 分评价四项：

1. 变量及变量组合是否具有领域意义；
2. 结构是否符合基本科学常识、是否存在明显牵强关系；
3. 公式是否易读、易概括；
4. 是否能据此提出可讨论、可检验的假设。

四项均值为 `Interpretability Score`，但必须同时给出各维度结果。

### 4.2 人类评分

- 至少 2 位领域评分者，能有 3 位更好；
- 6 个真实任务，每个任务比较 4–6 个性能匹配公式；
- 预计每位评价约 30–36 个公式，而不是 60 对以上的大型调查；
- 公式统一逆标准化、完全展开、统一排版；
- 隐藏方法名、R²、生成说明和 LLM 解释；
- 随机公式顺序；
- 报告评分者间一致性和 bootstrap 置信区间。

### 4.3 LLM 评分

- 使用两个不同模型家族；
- 使用与人类完全相同的变量说明、公式和 rubric；
- 每个公式改变呈现顺序至少重复一次，检查位置偏差；
- 报告 LLM 分、人类分以及二者相关性；
- **不把人类分和 LLM 分平均成一个总分**；
- 用于评价的模型不得参与 IF-SR 的最终选式。

## 7. 砍项原则

以下工作本轮一律不做：

- 不扩展到多个真实科学领域；
- 不新建 100+ 公式 benchmark；
- 不训练新的 LLM judge 或人类偏好模型；
- 不重写 PySR 或开发新的遗传编程引擎；
- 不接入超过一个外部 SR 框架；
- 不做大规模专家调查；
- 不把 LLM 生成的解释文本当作公式可解释性的证据；
- 不把节点数直接叫作“可解释性”；
- 不声称从真实噪声数据中恢复了真实物理定律或因果机制；
- 不因结果不好临时更换主任务、split、rubric 或 ExprSim 权重。

优先砍 P1 扩展：额外数据集、额外 baseline、更多噪声档、更多 seed、额外 LLM、复杂稳定性指标。不能砍：无泄漏、独立 test、公平预算、至少 2 位专家、两个实验和完整失败报告。

## 8. Go / No-Go 标准

08-10 做第一次判断，满足下列条件就继续投 IEEE BigData，不再无限加实验：

- [ ] 两个实验的主运行成功率至少 90%，失败原因可解释；
- [ ] IF-SR 相对最强 baseline 的真实任务中位 `ΔR² ≥ -0.02`；
- [ ] IF-SR 的人类可解释性均分或成对偏好稳定高于标准 PySR；
- [ ] 两个 LLM judge 至少在方向上与人类结论一致；
- [ ] 真式实验的 ExprSim 或等价恢复率至少一项稳定提升；
- [ ] 所有方法使用相同 split 和可解释的公平预算；
- [ ] 论文所有核心主张都有对应主表或统计检验。

若出现以下情况，按预设降级，不临时包装：

- R² 非劣效成立、可解释性提升：按当前主线投稿；
- 可解释性提升但 R² 明显下降：写成明确的 accuracy–interpretability trade-off，不声称全面优于；
- 人类提升但 LLM 不一致：以人类为主结果，把 LLM judge 的局限作为发现；
- 真式相似度不提升：真实数据论文仍可投，但删除“更好恢复结构”的贡献，实验 2 作为边界分析；
- 到 08-08 尚未完成主要运行或无两位专家：不提交残缺论文，立即切换到下一个 CCF C Full Paper 窗口。
